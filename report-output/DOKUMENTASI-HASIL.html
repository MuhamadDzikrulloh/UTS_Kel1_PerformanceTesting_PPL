1. Tujuan

  1. Melakukan pengujian performa sistem untuk mengukur waktu respon website saat diakses pengguna, khususnya dari proses login hingga halaman produk ditampilkan (Playwright).
  2. Melakukan load testing pada halaman utama (HTTP) dengan k6 untuk melihat perilaku di bawah beberapa pengguna virtual — melengkapi pengujian single-user.

2. Tools dan teknologi

ToolKegunaan
PlaywrightOtomatisasi browser; mengukur durasi end-to-end (login → produk).
Node.jsRuntime untuk menjalankan tes Playwright dan skrip npm.

DockerMenjalankan k6 tanpa instalasi binary lokal (grafana/k6).
Website ujiSauceDemo (https://www.saucedemo.com/)

3. Skenario pengujian

3.1 Playwright (satu pengguna)

  1. Membuka website SauceDemo
  2. Mengisi username dan password (standard_user / secret_sauce)
  3. Menekan tombol login
  4. Menunggu hingga halaman produk muncul (selector .inventory_list)

3.2 Load testing — k6

SauceDemo berupa SPA; login diproses di browser. Skenario load test memuat GET halaman utama berulang dengan banyak virtual user untuk mensimulasikan lalu lintas pengunjung. Ini melengkapi ukuran waktu satu pengguna di Playwright.

4. Langkah-langkah pengujian

a. Persiapan proyek:

cd performance-testing
npm ci
npx playwright install

b. File tes: tests/performance.spec.js. Inti pengukuran waktu mengikuti pola: Date.now() sebelum dan sesudah alur, lalu console.log('Total Time:', …, 'ms').

c. Menjalankan tes Playwright:

npx playwright test tests/performance.spec.js
npx playwright test --ui

Laporan HTML Playwright: npx playwright show-report

d. Load test (Docker menyala): npm run load atau npm run load:ci

e. Alur lengkap dokumentasi hasil: npm run perf:all

i. Analisis

Playwright (satu pengguna)

Berdasarkan pengujian Playwright pada SauceDemo, total waktu respons pada contoh dokumen referensi sekitar 778 ms tergolong cepat (di bawah 1 detik). Navigasi, input, login, dan muat halaman produk menunjukkan perilaku responsif untuk satu pengguna. Hasil terkini tercantum pada ringkasan eksekusi dan screenshot di atas.

Keterbatasan: skenario Playwright hanya mencerminkan satu pengguna; kondisi beban tinggi diwakili oleh pengujian k6.

Load testing (k6)

Pada skenario GET halaman utama dengan beberapa virtual user, metrik agregat (tabel di atas) menunjukkan perilaku HTTP di bawah threshold yang ditetapkan di skrip. Catatan metodologi: load test mengukur lalu lintas ke URL utama; alur login penuh di browser tetap pada bagian Playwright.

j. Kesimpulan

  • Playwright: skenario satu pengguna dari buka situs hingga halaman produk selesai dalam 847 ms, menunjukkan alur end-to-end berjalan dalam waktu yang terukur.
  • k6: load test membebani URL utama dengan beberapa virtual user; metrik utama tercantum pada tabel. Threshold di skrip: p95 < 5000 ms, error rate < 5%, checks > 95%.
  • Catatan: SauceDemo berbasis SPA — load test HTTP mengukur halaman entry; login penuh diwakili oleh Playwright.
